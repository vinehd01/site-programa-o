2. Na aula 08 Repetição otimizada com For é realizada a mudança do while para for. Porque? o que mudou? Explique.

A estrutura de repetição while depende de uma condição ser verdadeira ou falsa para determinar se aquela repetição, essa rotina que está dentro das chaves do while vai ser aplicada, que no caso, era o contador. Porém, na situação da aula, se programássemos instrumentos que possuem botões (ou seja, outros while’s), seria necessário declarar outros contadores, o que ficaria redundante. Para evitar esse problema, utilizamos a estrutura for. O for é uma estrutura de laço de repetição ou de loop, que também vai fazer a mesma coisa, enquanto uma instrução, uma condição for verdade, ele vai repetir essa rotina, porém a vantagem em relação ao while é que o contador pode ser declarado ainda dentro dos parênteses do for.

3. Na aula 09 Melhorando o código do Alura quais foram as mudanças realizadas e quais as condições adicionadas?

Foi reduzida a quantidade de if’s, usando o operador lógico ‘&’ (‘e’ em português), o else também foi adicionado como uma estrutura conjunta se a condição for falsa. O else significa “senão” e ele precisa ser invocado imediatamente depois de um if, que vai ser a função que vai ser executada caso esta condição que seja falsa. E então se o elemento não for nulo e o elemento for diferente do áudio, irá cair no else. Diminuir (--)

O operador decremento ( --) decrementa (subtrai um) seu operando e retorna o valor antes ou depois do decremento, dependendo de onde o operador é colocado. let x = 3; const y = x--; // x is 2; y is 3 let x2 = 3n; const y2 = x2--; // x2 is 2n; y2 i

Atribuição de divisão (/=)

O operador de atribuição de divisão ( /=) realiza a divisão nos dois operandos e atribui o resultado ao operando esquerdo. let bar = 5; bar /= 2; // 2.5 bar /= 2; // 1.25 bar /= 0; // Infinity bar /= "foo"; // NaN let foo = 3n; foo /= 2n; // 1n foo /= 2n; // 0n

Igualdade (==)

O operador igualdade ( ==) verifica se seus dois operandos são iguais, retornando um resultado booleano. Ao contrário do operador de igualdade estrita , ele tenta converter e comparar operandos de tipos diferentes. Ex: console.log(1 == 1); // Expected output: true console.log('hello' == 'hello'); // Expected output: true console.log('1' == 1); // Expected output: true console.log(0 == false); // Expected output: true

iogualdade estrita (===)

Retorna verdadeiro caso os operandos sejam iguais e do mesmo tipo. Veja também Object.is e igualdade em JS (en- US). Ex: 3 === var1

Desigualdade estrita (!==)

Retorna verdadeiro caso os operandos não sejam iguais e/ou não sejam do mesmo tipo. Ex: var1 !== "3" 3 !==

Subtração (-)

Operador uniário. Retorna a negação de seu operando. Ex: Se x é 3, então -x retorna -3.

Atribuição de subtração (-=)

O operador de atribuição de subtração (-=) realiza a subtração nos dois operandos e atribui o resultado ao operando esquerdo. let a = 2; console.log((a -= 3)); // Expected output: -1 console.log((a -= 'Hello')); // Expected output: NaN

O operador de exponenciação** ( ) eleva o primeiro operando à potência do segundo operando. let x = 5; let z = x ** 2;

Atribuição de exponenciação (**=)

Lógico NÃO (!)

Sempre que colocamos esse operador perto de alguma condição, ele altera o valor dela de verdadeiro para falso (true para false) ou o contrário, de falso para verdadeiro. if(true){ [esse codigo sempre executa]; } if(!true){ [esse codigo nunca executa]; }

OU lógico (||)

Unimos mais de uma expressão, usando o operador &&. Se a condição1 OU condição2 for verdadeira, todo o teste condicional será verdadeiro. Atribuição OR lógica (||=)

O operador de atribuição OR lógico é usado entre dois valores. Se o primeiro valor for falso, o segundo valor será atribuído. let x = 10; x ||= 5;
